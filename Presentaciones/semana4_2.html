Tipos de Relaciones entre Entidades en Bases de Datos

En este documento se explican los tipos de relaciones entre entidades que se utilizan comúnmente en bases de datos relacionales, con ejemplos prácticos.

1. Relación Uno a Uno (1:1)

En esta relación, una entidad A está asociada con una y solo una entidad B, y viceversa.

-- Ejemplo en SQL CREATE TABLE personas ( id INT PRIMARY KEY, nombre VARCHAR(100) ); CREATE TABLE pasaportes ( id INT PRIMARY KEY, numero_pasaporte VARCHAR(50), id_persona INT UNIQUE, FOREIGN KEY (id_persona) REFERENCES personas(id) );

Ejemplo práctico: Una persona tiene un único pasaporte, y un pasaporte pertenece a una sola persona.

2. Relación Uno a Muchos (1:N)

En esta relación, una entidad A puede estar asociada con múltiples entidades B, pero cada entidad B está asociada con solo una entidad A.

-- Ejemplo en SQL CREATE TABLE categorias ( id INT PRIMARY KEY, nombre VARCHAR(100) ); CREATE TABLE productos ( id INT PRIMARY KEY, nombre VARCHAR(100), id_categoria INT, FOREIGN KEY (id_categoria) REFERENCES categorias(id) );

Ejemplo práctico: Una categoría puede tener varios productos, pero cada producto pertenece a una sola categoría.

3. Relación Muchos a Muchos (N:M)

En esta relación, múltiples entidades A pueden estar asociadas con múltiples entidades B. Generalmente se implementa con una tabla intermedia.

-- Ejemplo en SQL CREATE TABLE estudiantes ( id INT PRIMARY KEY, nombre VARCHAR(100) ); CREATE TABLE cursos ( id INT PRIMARY KEY, nombre VARCHAR(100) ); CREATE TABLE estudiantes_cursos ( id_estudiante INT, id_curso INT, PRIMARY KEY (id_estudiante, id_curso), FOREIGN KEY (id_estudiante) REFERENCES estudiantes(id), FOREIGN KEY (id_curso) REFERENCES cursos(id) );

Ejemplo práctico: Un estudiante puede inscribirse en varios cursos, y un curso puede tener varios estudiantes inscritos.
