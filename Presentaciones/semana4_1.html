Ejercicio de Base de Datos

Sistema de Gestión de Hospital

Diseña una base de datos para un sistema de gestión de un hospital que permita organizar la información de pacientes, médicos, tratamientos, citas, medicamentos y facturación. La base de datos debe cumplir con las siguientes especificaciones:

Tablas Requeridas

1. Tabla Pacientes

  • idPaciente (clave primaria)
  • nombreCompleto
  • fechaNacimiento
  • direccion
  • telefono
  • email

2. Tabla Medicos

  • idMedico (clave primaria)
  • nombreCompleto
  • especialidad
  • telefono
  • email

3. Tabla Citas

  • idCita (clave primaria)
  • idPaciente (clave foránea de la tabla Pacientes)
  • idMedico (clave foránea de la tabla Medicos)
  • fechaCita
  • horaCita
  • motivo

4. Tabla Tratamientos

  • idTratamiento (clave primaria)
  • idCita (clave foránea de la tabla Citas)
  • descripcion
  • fechaInicio
  • fechaFin

5. Tabla Medicamentos

  • idMedicamento (clave primaria)
  • nombre
  • descripcion
  • precioUnidad

6. Tabla Recetas

  • idReceta (clave primaria)
  • idTratamiento (clave foránea de la tabla Tratamientos)
  • idMedicamento (clave foránea de la tabla Medicamentos)
  • cantidad
  • dosis

7. Tabla Facturas

  • idFactura (clave primaria)
  • idCita (clave foránea de la tabla Citas)
  • total
  • fechaFactura

Objetivo del Ejercicio

Diseñar las tablas cumpliendo con las especificaciones mencionadas, establecer las claves primarias y foráneas correctamente, normalizar las tablas hasta la tercera forma normal (3FN) y generar consultas SQL para:

  • Obtener todos los tratamientos de un paciente en un rango de fechas.
  • Listar las citas de un médico específico en una semana.
  • Calcular el total de medicamentos recetados por tratamiento.
  • Generar un reporte de facturación mensual.
