Funciones de Agregación en SQL

Explora cómo usar GROUP BY y HAVING para analizar datos agrupados.

¿Qué son las funciones de agregación?

Las funciones de agregación en SQL permiten realizar cálculos matemáticos o estadísticas sobre un conjunto de datos. Estas funciones son esenciales para analizar información en bases de datos.

  • COUNT: Cuenta el número de registros.
  • SUM: Suma los valores de una columna.
  • AVG: Calcula el promedio de los valores.
  • MIN: Encuentra el valor mínimo.
  • MAX: Encuentra el valor máximo.

Uso de GROUP BY

La cláusula GROUP BY se utiliza para agrupar filas que tienen valores similares en una o más columnas. Es especialmente útil para aplicar funciones de agregación sobre grupos de datos.

SELECT departamento, COUNT(*) AS total_empleados FROM empleados GROUP BY departamento;

Ejemplo práctico: Contar el número de empleados por departamento.

Uso de HAVING

La cláusula HAVING permite filtrar los resultados después de aplicar la agrupación con GROUP BY. Es similar a WHERE, pero se usa con funciones de agregación.

SELECT departamento, AVG(salario) AS salario_promedio FROM empleados GROUP BY departamento HAVING AVG(salario) > 5000;

Ejemplo práctico: Mostrar solo los departamentos con un salario promedio mayor a 5000.

Comparación entre WHERE y HAVING

WHERE: Filtra filas antes de aplicar funciones de agregación.
HAVING: Filtra los resultados después de aplicar las funciones de agregación.

-- Usando WHERE SELECT * FROM empleados WHERE salario > 5000; -- Usando HAVING SELECT departamento, AVG(salario) AS salario_promedio FROM empleados GROUP BY departamento HAVING AVG(salario) > 5000;

Beneficios de las funciones de agregación

  • Permiten obtener información resumida de grandes conjuntos de datos.
  • Facilitan la creación de informes analíticos.
  • Ayudan a identificar patrones y tendencias en los datos.
